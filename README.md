    📖 О проекте
BTC Bloom Scanner — это профессиональный инструмент для массового сканирования адресов Bitcoin.
Он использует технологию Bloom Filter (Блум-фильтр) для быстрой предварительной проверки адресов и API для точной проверки баланса. Все управление осуществляется через удобный веб-интерфейс.
    💻 Системные требования
ОС: Windows 10/11
Python: Строго версия 3.12.x (не 3.13 или 3.14!)
Интернет: Для работы API проверок
Файлы:
        addresses.bloom (обязательно должен быть в папке со скриптами)
        Его скачивайте по этой ссылке (https://disk.yandex.ru/d/GCQEqxEEo81v9w)
        И просто помещайте в папку (BloomScanner)
        Все файлы проекта (.py, папки templates, static)
    🚀 Установка (Пошагово)
Шаг 1: Установка Python 3.12
Скачайте Python 3.12 с официального сайта python.org.
При установке обязательно поставьте галочку ✅ "Add Python to PATH".
Завершите установку.
Шаг 2: Установка библиотек
Откройте командную строку (CMD) или PowerShell в папке с проектом и выполните:
cmd
            py -3.12 -m pip install bit mnemonic requests tqdm flask flask-socketio
Это установит все необходимые модули для работы генерации ключей и веб-интерфейса.
Шаг 3: Подготовка Bloom Filter
Убедитесь, что файл addresses.bloom лежит в той же папке, где и scanner.py. Без него сканер не запустится.
Запуск Сканера
Для запуска не нужно писать код в консоли. Мы создали специальный Батник.
Найдите файл start_server.bat в папке проекта.
Запустите его двойным кликом.
Откроется чёрное окно (консоль сервера). Дождитесь надписи:
            🌐 GUI: http://localhost:5000
Откройте браузер (Chrome/Edge) и перейдите по адресу:
👉 http://localhost:5000
    🎮 Интерфейс: Что за что отвечает
1. 📊 Панель статистики (Верхняя часть)
Generated: Количество сгенерированных приватных ключей.
Hits: Количество "подозрительных" адресов, найденных Блум-фильтром (ложные срабатывания + реальные).
Addr/sec: Скорость генерации (адресов в секунду).
API Checked: Сколько адресов из списка Hits было проверено через интернет.
2. 🏆 Found with Balance (Красная панель)
Здесь появляются кошельки, у которых найден баланс > 0.
Счётчик справа показывает общее количество таких кошельков.
3. ⚙️ Configuration (Настройки)
Telegram Notifications:
Вставьте Bot Token и Chat ID, включите тумблер, чтобы получать уведомления о найденных балансах на телефон.
Method: Метод хеширования (оставьте pubkey_hash160 для стандарта Bitcoin).
Threads: Количество потоков процессора (рекомендуется 6-8).
Target: Цель сканирования (сколько адресов сгенерировать).
Кнопки:
🟢 Start: Начать сканирование.
⏸ Pause: Поставить на паузу (сохраняет прогресс).
🛑 Stop: Полная остановка скрипта.
4. 📈 Speed & Console (Нижняя часть)
Speed: График показывает нагрузку и скорость работы в реальном времени.
Console: Лог работы (показывает ошибки, успешные запуски, действия API воркера).
    🧠 Как это работает (Логика)
Генерация: Скрипт создает случайные мнемонические фразы и приватные ключи.
Блум-фильтр: Быстро проверяет адрес по загруженной базе.
Если адрес НЕ в базе: Скрипт переходит к следующему.
Если адрес ЕСТЬ в базе (Hit): Адрес отправляется в очередь API.
API Воркер: Берет адрес из очереди и стучится в блокчейн (blockstream.info).
Важно: Эта проверка медленная (20-50 сек) для безопасности, чтобы не получить бан.
Результат: Если баланс найден, он отображается в красной панели и отправляется в Telegram.
    ⚠️ Возможные проблемы
❌ Ошибка "No module named 'bit'"
Решение: Запустите установку библиотек заново, убедившись, что используете Python 3.12:
py -3.12 -m pip install bit mnemonic requests tqdm flask flask-socketio
❌ "Disconnected" в углу экрана
Решение:
Проверьте, не закрыли ли вы чёрное окно консоли (сервер должен работать).
Обновите страницу в браузере через Ctrl + F5.
❌ API Checked не растет
Решение: Это нормально. API проверяет только те адреса, которые нашёл Блум-фильтр (Hits). Если Hits = 0, то и проверять нечего.
❌ Сканер завис или не останавливается
Решение: В новой версии исправлена кнопка Stop. Если всё равно зависло — закройте чёрное окно консоли (Ctrl+C) и запустите start_server.bat заново.
