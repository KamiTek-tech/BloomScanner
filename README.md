🛡️ BTC Bloom Scanner Pro
Высокопроизводительный инструмент для сканирования Bitcoin-пространства с каскадной Bloom-фильтрацией, встроенным модулем восстановления мнемонических фраз (Recovery) и современным веб-интерфейсом.

📖 Описание
Проект объединяет генератор BIP39-ключей, многопоточный сканер адресов и систему восстановления утерянных сид-фраз. Главная особенность — двухуровневая каскадная проверка, которая снижает нагрузку на внешние API на 99.9%+, оставляя для проверки только адреса с высокой вероятностью существования.
⚡ Ключевые возможности
🔹 Каскадная Bloom-фильтрация
Быстрый фильтр (~714 МБ, FP rate 1e-5) → Строгий фильтр (1e-7 или 1e-9, размер 1.2–4 ГБ) → API-проверка баланса. Ложные срабатывания сведены к минимуму.
🔹 Mnemonic Recovery Engine
Три режима восстановления:
❓ Missing Words – подбор 1–2 пропущенных слов.
🔀 Shuffled – перебор перемешанного порядка (до 9 слов).
🔤 Typo Fix – исправление опечаток с учётом уникальных префиксов BIP39.
🔹 Умная API-интеграция
Рандомные задержки (20–50 сек для сканера, 2–5 сек для Recovery), защита от банов, автоматическая проверка баланса через blockstream.info.
🔹 Telegram-уведомления
Мгновенные сообщения о найденных кошельках с балансом > 0 BTC.
🔹 Автоэкспорт данных
Все находки сохраняются в results/found_wallets.txt и SQLite БД (results/scan_*.db).
🔹 Веб-интерфейс с синхронизацией
Графики скорости, консоль логов, панель баланса, табы Recovery. GUI продолжает обновляться после завершения генерации, пока API-воркер обрабатывает очередь.
🔹 UI-бейдж Created at kamiteʞ
Анимированный градиентный индикатор в шапке интерфейса.

🏗 Архитектура и логика работы
🔍 Каскадная проверка (Cascade Filtering)
    Генерация адреса → Быстрый Bloom (отсеивает 99.9%) → Строгий Bloom (отсеивает 99.999%) → API Check → Экспорт/Telegram
Если быстрый фильтр возвращает False → адрес игнорируется мгновенно.
Если проходит оба фильтра → ставится в очередь API.
API подтверждает баланс → данные сохраняются и отправляются в Telegram.

🔑 Recovery Mode
Валидация BIP39 контрольной суммы (отсеивает 99.9% невалидных комбинаций).
Деривация адреса (m/44'/0'/0'/0) и расчёт Hash160.
Проверка через Bloom-фильтр.
Финальная верификация баланса через API.
Уведомление + сохранение в файл при balance > 0.

🌐 Веб-сервер
Backend: Flask + Flask-SocketIO (асинхронная передача событий).
Frontend: Bootstrap 5 + Chart.js + кастомные CSS-анимации.
Потоки: Генерация, API-воркер, обновление статистики и Recovery работают в изолированных daemon-потоках, не блокируя GUI.

📦 Установка
1. Требования
Python строго 3.12
Git (опционально)
2. Зависимости
    pip install bit mnemonic requests tqdm flask flask-socketio
3. Bloom-фильтры
Для работы требуется два файла в корне проекта:
addresses.bloom – быстрый фильтр (грубая проверка). Скачать тут https://disk.yandex.ru/d/GCQEqxEEo81v9w
addresses_strict.bloom – строгий фильтр (точная проверка). Скачать тут https://disk.yandex.ru/d/CN6R8xHNPchddg или создать свой
📥 Генерация строгого фильтра: python create_strict_bloom.py
Скрипт читает архив all_Bitcoin_addresses_ever_used_sorted.txt.gz и создаёт фильтр с настраиваемым ERROR_RATE. Процесс занимает от 12 до 48 часов в зависимости от размера архива и мощности CPU.

🚀 Запуск и использование
1. Запуск сервера
    start_server.bat   # Windows
# или
python web_gui.py
Откройте в браузере: http://localhost:5000
2. Конфигурация сканера
Method: pubkey_hash160 (рекомендуется) или address_sha256
Threads: количество потоков генерации (1–32)
Target: лимит генерации для текущей сессии
Включите Telegram, указав Bot Token и Chat ID
3. Использование Recovery
Перейдите во вкладку Mnemonic Recovery.
Выберите режим: Missing Words, Shuffled или Typo Fix.
Введите данные согласно подсказкам (используйте ? для пропусков).
Нажмите 🔍 Начать поиск. Прогресс и найденные фразы отображаются в реальном времени.

⚙️ Настройка Telegram и API
Telegram: Данные передаются из GUI в переменные окружения TG_ENABLED, TG_TOKEN, TG_CHAT_ID. Применяются к обоим режимам (Scanner и Recovery).
API задержки:
Scanner: random.uniform(20, 50) сек между запросами.
Recovery: random.uniform(2, 5) сек (оптимизировано для перебора).
Лимиты Recovery:
Missing: до 2 слов (2048² ≈ 4.1 млн комбинаций)
Shuffled: до 9 слов (9! = 362 880 перестановок)
Typo: до 3-4 слов с опечатками (комбинаторика < 50 000)

🛡 Безопасность и ограничения
🔒 Все данные обрабатываются локально. Сид-фразы не передаются на сторонние серверы.
💾 Потребление RAM зависит от размера строгого фильтра: ~700 МБ (1e-5) → ~4 ГБ (1e-7 + 1 млрд capacity).
⚠️ Recovery не гарантирует восстановление при потере >3 слов или полном отсутствии данных. Используйте только для собственных кошельков.
